iTrust - Semester Winner
NCSU - CSC 326 - Software Engineering Class. Fall 2016
Overview:
Over the course of a semester in Software Engineerings (CSC 326) our team performed improvements on iTrust - a program that has been worked on by previous CSC students at NCSU over the past decade. Our team competed with other teams in the class where we all completed the same requirements in order to create the best iTrust project for that semester. At the end of the semester our project was selected as best in the class after we presented our project in front of both our lab and our entire class.

Goal:
Use JSF (Java Server Framework) to implement five additional use cases, refactor logging, secure SQL connections with prepared statements, implement mocking through Mockito for proper unit testing, and create a docker image.

Process:
The process involved 5 iterations where our team worked in an agile environment. We would meet twice a week and have stand up meetings to describe where we were on the project and what help, if any, was needed. We managed the project through GitHub where we decided to utiulze merge requests to code review our merges and GitHub issues that we could close as we finished our assignments.

Door To More
Off The Grid : Blackbaud. Summer 2016 - finalist
Door To More was created in 24 hours to benefit advocacy organizations that would employ individuals to canvass door to door or on the street to raise awareness, soulcit donations, and add to in-house files. The purpose of the app is to create a streamulned interface where canvassers can quickly create new constituents and document the interaction in a way that advocacy organizations can aggregate or target these constituents in the future.

Research File Anonymizer
Created by Thomas Ortiz. Spring 2016
This java program was developed to quickly anonymize student files by removing identifying information such as email, names, and @author tags and replacing them with encrypted ids generated by the research director. As the program parses through student files it creates new anonymized files and saves the new files into a directory called Anon_Submissions while preserving the old copies of the files.
