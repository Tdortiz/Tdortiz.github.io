Blackbaud - Summer 2017
During my second internship at Blackbaud I worked as a full-stack developer on their web platform Financial Edge Nxt (FEnxt). I worked with a scrum team performing bi-weekly sprints to continuously integrate new features into FEnxt.

Specifically, my intern project was working in C# .net for backend work and then developing with Angular 1.x for the frontend. The project's goal was to add functionality for the user to see the status of a process users can trigger. Before this project users would kick off the process and it would finish without specific feedback past whether or not the process worked or didin't from a tiny toast message. Once integrated, my tasks allowed users to see how and why processes succeeded or failed and provided a history log of each process.

After finishing my project I moved on to other sprint tasks with my team. Some examples of tasks I completed include optimizing end point tests, bug fixing across the product, and additional feature implementations.

During this internship I got more freedom to develop across the entire stack. I used C#, SQL, Javascript, jQuery, Angular 1.x, and other web technologies but what I was most interested in learning was the entire stack because I got a better understanding on how web applications work.

Jenkins Dashboard
NCSU - CSC Research - Spring/Summer 2017 Problem:
Computer Science courses use Jenkins to automate the build process and grading in many CSC classes. Student can see this output but the useful output is cluttered across Jenkin's website making it either hard to read or difficult to find out where to look.

Solution:
Our solution is to extend and utilize the data collection our team was already doing through build scripts and display all the neccessary project output information in a consolidated format.

Outcome:
Using Django, we built a dashboard for Teaching Assistant and students to easily look at build output. While originally this project was for one class, Java II, my team and other teachers saw the value in this project and as such we built the project to be easily modifiable depending on what class needs it. The current plans for this project is to use it in all classes in NCSU's CSC program that uses Jenkins to automate builds.

Learning Outcomes:
Back-End Development, Front-End Development, Django, Jenkins, SQL
iTrust - Class Winner
NCSU - CSC 326 - Software Engineering Class. Fall 2016 Overview:
Over the course of a semester in Software Engineerings (CSC 326) our team performed improvements on iTrust - a program that has been worked on by previous CSC students at NCSU over the past decade. Our team competed with other teams in the class where we all completed the same requirements in order to create the best iTrust project for that semester. At the end of the semester our project was selected as best in the class after we presented our project in front of both our lab and our entire class.

Goal:
Use JSF (JavaServer Faces) to implement five additional use cases, refactor logging, secure SQL connections with prepared statements, implement mocking through Mockito for proper unit testing, and create a docker image.

Process:
The process involved 5 iterations where our team worked in an agile environment. We would meet twice a week and have stand up meetings to describe where we were on the project and what help, if any, was needed. We managed the project through GitHub where we decided to utilize merge requests to code review our merges and GitHub issues that we could close as we finished our assignments.
